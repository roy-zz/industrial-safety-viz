### HAZOP(위험과 운전분석) 기법의 정의와 3대 구성요소를 설명하시오. ·Q15-3
- 🔖 빈출: 예상
- 📌 근거: KOSHA C-C-37-2026 · PSM 고시 제2조
- 📖 모범답안 — KOSHA C-C-37 제3조 + PSM 고시 제2조제11호:
  - [C-C-37 3조(가)] "위험과 운전분석(HAZOP study)"이라 함은 공정에 존재하는 위험요인과 공정의 효율을 떨어뜨릴 수 있는 운전상의 문제점을 찾아내어 그 원인을 제거하는 방법을 말한다.
  - [C-C-37 3조] 변수(Parameter) = 유량·압력·온도 등 물리량이나 공정 흐름 조건을 나타내는 변수 / 가이드워드(Guide word) = 변수의 질이나 양을 표현하는 간단한 용어 / 이탈(Deviation) = 가이드워드와 변수가 조합되어 유체흐름의 정지·과잉상태와 같이 설계의도로부터 벗어난 상태.
- 🔗 🔗 함정(❌) HAZOP 핵심 = 이탈 = 가이드워드 × 변수. "위험요인 + 운전성(operability) 문제"를 함께 본다(안전+생산성). 연계(→Q15-4·Q15-5)
### HAZOP의 가이드워드 종류와 의미를 설명하시오. ·Q15-4
- 🔖 빈출: 실제기출 10회 (가이드워드)
- 📌 근거: KOSHA C-C-37-2026 표1
- 📖 모범답안 — KOSHA C-C-37 <표 1> 연속공정에 대한 가이드워드:
  - | 가이드워드 | 정의 |
  - |---|---|
  - | 없음(No/Not/None) | 설계의도에 완전히 반하여 변수의 양이 없는 상태 (예: 흐름없음 No flow) |
  - | 증가(More) | 변수가 양적으로 증가되는 상태 (예: 흐름증가 More flow) |
  - | 감소(Less) | 변수가 양적으로 감소되는 상태 (작은 경우 No로 표현 가능) |
  - | 반대(Reverse) | 설계의도와 정반대로 나타나는 상태 (예: 역류 Reverse flow) |
  - | 부가(As well as) | 설계의도 외에 다른 변수가 부가되는 상태 (예: 오염 Contamination) |
  - | 부분(Parts of) | 설계의도대로 완전히 이루어지지 않는 상태 (예: 조성비율 오류) |
  - | 기타(Other than) | 설계의도대로 설치·운전 유지되지 않는 상태 (예: 밸브 오설치, 다른 원료 공급) |
- 🔗 🔗 함정(❌) 7개 가이드워드 = No/More/Less/Reverse/As well as/Parts of/Other than. 변수(유량·압력·온도)와 조합 → 이탈 도출. 연계(→Q15-3)
- 💡 💡 암기: "무증감반부분기" 💥 무죄 증거 감춘 반장 — 부분 기소! — 무(없음 No/Not)·증(증가 More)·감(감소 Less)·반(반대 Reverse)·부(부가 As well as)·분(부분 Parts of)·기(기타 Other than)
### HAZOP의 검토구간(Node) 설정과 평가 진행절차를 설명하시오. ·Q15-5
- 🔖 빈출: 예상
- 📌 근거: KOSHA C-C-37-2026
- 📖 모범답안 — KOSHA C-C-37 제5~6장:
  - [C-C-37 6.2] 검토구간은 공정의 설계 목적(유체 흐름방향·온도·압력·액위의 증감 등)과 공정의 복잡성(화학반응·제어 논리 등)에 따라 구분하고, 공정의 복잡성(배관 수량 등) 및 위험성평가 팀의 경험에 따라 그 크기를 결정한다.
  - [C-C-37 6.3] 검토구간을 정하고 설계목적·특성을 설명한 후 <표 1>의 가이드워드와 변수를 조합한 이탈을 도출하여 정상운전 상태로부터 벗어날 수 있는 가능한 원인과 결과를 조사한다. 평가가 끝난 구간은 녹색, 수정·변경 필요 시 적색으로 표시한다. 과거 유사설비의 중대산업사고·공정사고·아차사고도 평가한다.
- 🔗 🔗 함정(❌) Node = P&ID상 검토구간. 절차: 구간 선정 → 설계의도 설명 → 가이드워드×변수=이탈 → 원인·결과·기존안전조치 → 개선권고. 색 구분(완료=녹/수정=적). 연계(→Q15-3·Q15-4)

### 결함수분석(FTA) 기법의 정의·특징과 주요 기호를 설명하시오. ·Q15-10
- 🔖 빈출: 예상
- 📌 근거: PSM 고시 제2조 · KOSHA C-C-39-2026
- 📖 모범답안 — PSM 고시 제2조제13호 + KOSHA C-C-39:
  - [고시 제2조13호] "결함수분석(Fault Tree Analysis, FTA)기법"이란 사고의 원인이 되는 장치의 이상이나 고장의 다양한 조합 및 작업자 실수 원인을 연역적으로 분석하는 방법을 말한다.
  - [C-C-39 9.1] 공정의 기능상실을 정상 사상(Top event)으로 정의하고 그러한 정상 사상이 발생할 수 있는 원인과 경로를 연역적으로 분석한다. 절차: 정상사상 정의 → 결함수 구성 → 최소컷세트 → 정성분석 → (정량화: 불확실성·중요도 분석).
  - [C-C-39 표1 기호] 기본사상(○)·중간사상(△)·생략사상(◇)·통상사상(⌂)·조건부사상(⊖) / OR 게이트(⊕) = 한 개 이상 입력사상 발생 시 출력 발생 / AND 게이트(⊗) = 입력사상이 전부 발생해야 출력 발생 / 억제 게이트(⊘) = AND의 특별형(특정조건 만족 필요).
- 🔗 🔗 함정(❌) FTA = 하향식·연역(결과→원인), 정상사상에서 출발. OR=합집합(어느 하나)/AND=교집합(모두). 연계(→Q15-12·Q15-13)
### 사건수분석(ETA) 기법을 설명하시오. ·Q15-11
- 🔖 빈출: 예상
- 📌 근거: PSM 고시 제2조 · 통설
- 📖 모범답안 — PSM 고시 제2조제14호 + [통설]:
  - [고시 제2조14호] "사건수분석(Event Tree Analysis, ETA)기법"이란 초기사건(Initiating event)으로 알려진 특정한 장치의 이상 또는 운전자의 실수에 의해 발생되는 잠재적인 사고결과를 정량(定量)적으로 평가ㆍ분석하는 방법을 말한다.
  - [통설] 하나의 초기사건에서 출발해 안전장치·방호계층의 성공/실패 분기를 따라가며 가능한 결과 시나리오와 각 빈도를 산정. 귀납적·전향적(forward)·상향식 사고전개.
- 🔗 🔗 함정(❌) ETA = 귀납·전향(원인→결과), 초기사건에서 출발. FTA(연역·후향)와 짝. 둘을 결합 = Bow-tie(나비넥타이) 분석. 연계(→Q15-10·Q15-12)
### FTA와 ETA의 차이를 비교하여 설명하시오. ·Q15-12
- 🔖 빈출: 예상
- 📌 근거: PSM 고시 제2조 · KOSHA C-C-39-2026 · 통설
- 📖 모범답안 — PSM 고시 제2조제13·14호 + [통설]:
  - | 구분 | FTA(결함수) | ETA(사건수) |
  - |---|---|---|
  - | 출발점 | 정상사상(Top event, 사고결과) | 초기사건(Initiating event, 원인) |
  - | 방향 | 연역적·하향식(결과→원인) | 귀납적·전향식(원인→결과) |
  - | 논리 | AND/OR 게이트, 최소컷세트 | 안전장치 성공/실패 분기 |
  - | 산출 | 정상사상 발생확률·기여원인 | 결과 시나리오별 빈도 |
  - [통설] FTA는 "이 사고는 왜 일어나는가", ETA는 "이 초기사건이 어떤 결과로 이어지는가". 두 기법을 Bow-tie로 결합하면 원인(좌)–정상사상(중앙)–결과(우)를 한눈에 본다.
- 🔗 🔗 함정(❌) FTA=연역/후향, ETA=귀납/전향(혼동 빈출). FMECA·HEA도 귀납계열(→Q15-9). 연계(→Q15-10·Q15-11·Q15-13)
### FTA의 최소컷세트(Minimal Cut Set)와 패스세트의 의미를 설명하시오. ·Q15-13
- 🔖 빈출: 예상
- 📌 근거: KOSHA C-C-39-2026 · 통설
- 📖 모범답안 — KOSHA C-C-39 제3조 + [통설]:
  - [C-C-39 3조(마)·(바)] "컷세트(Cutset)"라 함은 정상 사상을 발생시키는 기본 사상의 집합을 말한다. "최소 컷세트(Minimal cutset)"라 함은 정상 사상을 발생시키는 기본 사상의 최소집합을 말한다.
  - [통설] 최소컷세트 = 그 안의 기본사상이 모두 일어나면 반드시 정상사상이 발생하는, 더 줄일 수 없는 조합 → 시스템의 위험 경로(취약점). 패스세트(Path set) = 그 기본사상이 모두 일어나지 않으면 정상사상이 발생하지 않는 집합 → 안전(신뢰성) 경로. 최소컷세트가 적고 차수가 낮을수록(단일사상 컷세트 존재) 위험이 크다.
- 🔗 🔗 함정(❌) 컷세트=고장 경로, 패스세트=정상 경로. 최소컷세트의 차수가 1이면 단일고장으로 사고 → 우선 개선 대상. 연계(→Q15-10)

## 🔴 안전설계·재해이론
### Fail Safe, Fool Proof, Tamper Proof의 개념을 설명하시오. ·Q15-20
- 🔖 빈출: 실제기출 12회 (Fail Safe·Fool Proof·Tamper Proof)
- 📌 근거: 통설
- 📖 모범답안 — [통설] 본질적 안전설계 원칙:
  - Fail Safe(페일 세이프) = 부품·시스템이 고장 나더라도 안전측으로 작동하도록 설계하는 원칙. 예) 전원 차단 시 밸브가 닫히는 Fail-close, 신호 상실 시 정지. 유형: Fail-passive(정지)·Fail-active(경보 후 일정시간 운전)·Fail-operational(다음 점검까지 기능 유지).
  - Fool Proof(풀 프루프) = 작업자가 실수(오조작)해도 사고로 이어지지 않도록 하는 설계. 예) 안전커버 없으면 작동 불가, 양수(兩手) 조작 버튼, 인터록, 커넥터 오삽입 방지 형상.
  - Tamper Proof(탬퍼 프루프) = 안전장치를 임의로 해제ㆍ변경ㆍ무력화하지 못하도록 하는 설계. 예) 봉인, 공구 없이 해제 불가, 안전밸브 임의조정 방지.
  - Fool Proof 종류 5 = ① 가드(Guard): 조절 가드ㆍ경고 가드ㆍ인터록 가드 ② 인터록 장치: 기계식ㆍ전기식ㆍ유압식 ③ 트립(Trip): 신체 일부가 위험영역에 접근하면 기계 정지 ④ 오버런(Over-run): 스위치를 끈 후 관성운동ㆍ잔류전하를 감지하여 위험이 있는 동안 가드가 열리지 않음 ⑤ 밀어내기(Push-away): 가드 가동부가 열렸을 때 자동으로 비위험영역으로 신체를 밀어냄.
  - Fail Safe 예시 = 승강기 정전ㆍ정격속도 초과 시 조속기 작동 긴급정지, 장치의 Redundancy(다중화) 개념, Bypass Line 설치, 석유난로 기울어짐 시 자동소화, 철도신호 고장 시 적색신호 전환.
  - Tamper Proof 예시 = 휴즈(Fuse), (해제 시 작동 불능) 안전장치 봉인.
- 💡 💡 암기: Fool Proof 5 "가인트오밀" 💥 가인(歌人)이 트로트 오디션 밀어붙여 — 가(가드)·인(인터록)·트(트립)·오(오버런)·밀(밀어내기)
- 🔗 🔗 함정(❌) Fail Safe = 고장 시 안전측, Fool Proof = 사람 실수 방지, Tamper Proof = 임의 조작 방지(셋의 대상이 다름: 고장/실수/고의해제). 연계(→Q15-22 / 분류1 안전장치)

### Q15-10. 결함수분석(FTA) 기법의 정의·특징과 주요 기호를 설명하시오.
【빈출/회차】 예상
【근거】 [PSM 고시 제2조](https://www.law.go.kr/행정규칙/공정안전보고서의제출·심사·확인및이행상태평가등에관한규정) · [KOSHA C-C-39-2026]

**모범답안** — PSM 고시 제2조제13호 + KOSHA C-C-39:
> [고시 제2조13호] "**결함수분석(Fault Tree Analysis, FTA)기법**"이란 사고의 원인이 되는 장치의 이상이나 고장의 **다양한 조합 및 작업자 실수 원인을 연역적으로 분석**하는 방법을 말한다.
> [C-C-39 9.1] 공정의 **기능상실을 정상 사상(Top event)으로 정의**하고 그러한 정상 사상이 발생할 수 있는 원인과 경로를 **연역적으로 분석**한다. 절차: 정상사상 정의 → 결함수 구성 → **최소컷세트** → 정성분석 → (정량화: 불확실성·중요도 분석).
> [C-C-39 표1 기호] **기본사상(○)**·중간사상(△)·생략사상(◇)·통상사상(⌂)·조건부사상(⊖) / **OR 게이트(⊕)** = 한 개 이상 입력사상 발생 시 출력 발생 / **AND 게이트(⊗)** = 입력사상이 **전부 발생**해야 출력 발생 / **억제 게이트(⊘)** = AND의 특별형(특정조건 만족 필요).
> 🔗 함정(❌) FTA = **하향식·연역**(결과→원인), 정상사상에서 출발. OR=합집합(어느 하나)/AND=교집합(모두). 연계(→Q15-12·Q15-13)

### Q15-11. 사건수분석(ETA) 기법을 설명하시오.
【빈출/회차】 예상
【근거】 [PSM 고시 제2조](https://www.law.go.kr/행정규칙/공정안전보고서의제출·심사·확인및이행상태평가등에관한규정) · [통설]

**모범답안** — PSM 고시 제2조제14호 + [통설]:
> [고시 제2조14호] "**사건수분석(Event Tree Analysis, ETA)기법**"이란 **초기사건(Initiating event)**으로 알려진 특정한 장치의 이상 또는 운전자의 실수에 의해 발생되는 **잠재적인 사고결과를 정량(定量)적으로 평가ㆍ분석**하는 방법을 말한다.
> [통설] 하나의 **초기사건**에서 출발해 안전장치·방호계층의 **성공/실패 분기**를 따라가며 가능한 **결과 시나리오와 각 빈도**를 산정. **귀납적·전향적(forward)·상향식** 사고전개.
> 🔗 함정(❌) ETA = **귀납·전향**(원인→결과), 초기사건에서 출발. FTA(연역·후향)와 짝. 둘을 결합 = **Bow-tie**(나비넥타이) 분석. 연계(→Q15-10·Q15-12)

### Q15-12. FTA와 ETA의 차이를 비교하여 설명하시오.
【빈출/회차】 예상
【근거】 [PSM 고시 제2조](https://www.law.go.kr/행정규칙/공정안전보고서의제출·심사·확인및이행상태평가등에관한규정) · [KOSHA C-C-39-2026] · [통설]

**모범답안** — PSM 고시 제2조제13·14호 + [통설]:
> | 구분 | **FTA(결함수)** | **ETA(사건수)** |
> |---|---|---|
> | 출발점 | **정상사상**(Top event, 사고결과) | **초기사건**(Initiating event, 원인) |
> | 방향 | **연역적·하향식**(결과→원인) | **귀납적·전향식**(원인→결과) |
> | 논리 | AND/OR 게이트, **최소컷세트** | 안전장치 성공/실패 **분기** |
> | 산출 | 정상사상 발생확률·기여원인 | 결과 시나리오별 **빈도** |
> [통설] FTA는 "이 사고는 왜 일어나는가", ETA는 "이 초기사건이 어떤 결과로 이어지는가". 두 기법을 **Bow-tie**로 결합하면 원인(좌)–정상사상(중앙)–결과(우)를 한눈에 본다.
> 🔗 함정(❌) FTA=연역/후향, ETA=귀납/전향(혼동 빈출). FMECA·HEA도 귀납계열(→Q15-9). 연계(→Q15-10·Q15-11·Q15-13)

### Q15-13. FTA의 최소컷세트(Minimal Cut Set)와 패스세트의 의미를 설명하시오.
【빈출/회차】 예상
【근거】 [KOSHA C-C-39-2026] · [통설]

**모범답안** — KOSHA C-C-39 제3조 + [통설]:
> [C-C-39 3조(마)·(바)] "**컷세트(Cutset)**"라 함은 **정상 사상을 발생시키는 기본 사상의 집합**을 말한다. "**최소 컷세트(Minimal cutset)**"라 함은 **정상 사상을 발생시키는 기본 사상의 최소집합**을 말한다.
> [통설] **최소컷세트** = 그 안의 기본사상이 **모두 일어나면 반드시 정상사상이 발생**하는, 더 줄일 수 없는 조합 → **시스템의 위험 경로(취약점)**. **패스세트(Path set)** = 그 기본사상이 모두 일어나지 **않으면 정상사상이 발생하지 않는** 집합 → **안전(신뢰성) 경로**. 최소컷세트가 적고 차수가 낮을수록(단일사상 컷세트 존재) 위험이 크다.
> 🔗 함정(❌) **컷세트=고장 경로, 패스세트=정상 경로**. 최소컷세트의 차수가 1이면 **단일고장으로 사고** → 우선 개선 대상. 연계(→Q15-10)

### Q15-15. K-PSR과 HAZOP의 차이점을 설명하시오.
【빈출/회차】 예상
【근거】 [KOSHA C-C-61-2026](#k-cc61) · [KOSHA C-C-37-2026](#k-cc37)

**모범답안** — KOSHA C-C-61 + C-C-37:
> | 구분 | **HAZOP** | **K-PSR** |
> |---|---|---|
> | 대상 | 신·증설 포함 전 공정 | **기존 설치ㆍ가동중 공장 재검토** |
> | 질문 | 가이드워드 × 변수 = **이탈** | **위험형태 + 원인**(이중질문) |
> | 위험분류 | 이탈별 원인·결과 | **누출·화재폭발·공정트러블·상해 4형태** |
> | 소요 | 상세·장시간 | **간이·신속**(재검토 효율화) |
> [C-C-61 2장] K-PSR은 **HAZOP 등으로 평가한 후 다시 공정안전성을 재검토**하는 데 활용 → HAZOP을 대체하는 것이 아니라 **보완·후속 검토** 성격.
> 🔗 함정(❌) K-PSR ≠ HAZOP 대체. K-PSR은 **기존공장 간이 재검토**, 위험형태 4가지로 빠르게 스크리닝. 연계(→Q15-14·Q15-4)

【근거】 [PSM 고시 제2조](https://www.law.go.kr/행정규칙/공정안전보고서의제출·심사·확인및이행상태평가등에관한규정) · [통설]

**모범답안** — PSM 고시 제2조제15호 + [통설]:
> [고시 제2조15호] "**원인결과분석(Cause-Consequence Analysis, CCA)기법**"이란 잠재된 **사고의 결과 및 사고의 근본적인 원인을 찾아내고 사고결과와 원인 사이의 상호 관계를 예측**하여 위험성을 **정량(定量)적으로 평가**하는 방법을 말한다.
> [통설] **FTA(원인 분석)와 ETA(결과 분석)를 결합**한 기법 → 하나의 임계사건을 중심으로 원인측(FTA)과 결과측(ETA)을 동시에 전개. **Bow-tie**가 대표적 시각화.
> 🔗 함정(❌) CCA = **FTA + ETA 결합**(원인·결과 양방향, 정량). 연계(→Q15-10·Q15-11·Q15-12)

### Q15-17. 방호계층분석(LOPA)과 독립방호계층(IPL)을 설명하시오.
【빈출/회차】 예상
【근거】 [PSM 고시 제2조](https://www.law.go.kr/행정규칙/공정안전보고서의제출·심사·확인및이행상태평가등에관한규정) · [통설]

**모범답안** — PSM 고시 제2조제19호 + [통설]:
> [고시 제2조19호] "**방호계층 분석 기법(Layer of Protection Analysis, LOPA)**"이란 **사고의 빈도나 강도를 감소시키는 독립방호계층의 효과성을 평가**하는 방법을 말한다.
> [통설] **독립방호계층(IPL)** 요건 = ① **유효성**(해당 사고를 막을 수 있음) ② **독립성**(초기사건·다른 IPL과 무관) ③ **검증가능성**(시험·감사로 확인). 초기사건 빈도 × 각 IPL의 **요구시 고장확률(PFD)**을 곱해 **완화 후 사고빈도**가 허용기준 이내인지 판정. 계층 예: 공정설계 → 기본제어(BPCS) → 경보·운전원 대응 → **안전계장시스템(SIS)** → 물리적 방호(안전밸브·파열판) → 방유제·소화 → 비상대응.
> 🔗 함정(❌) LOPA = **반정량**(HAZOP과 정량평가 사이). IPL은 **독립·유효·검증가능** 3요건. SIL 결정과 연결. 연계(→Q15-18 / 분류1 안전밸브)
> 💡 **암기**: 방호계층 순서 "설제경계물방비" 💥 양파 방어 — **설**계가 심장, **제**어가 손발, **경**보 울리면 **계**장(SIS), **물**리 방호 뚫리면 **방**유제, 끝은 **비**상대응 [통설] — 설(공정설계)·제(기본제어 BPCS)·경(경보·운전원 대응)·계(안전계장 SIS)·물(물리적 방호: 안전밸브·파열판)·방(방유제·소화)·비(비상대응)

### Q15-18. 정량적 위험성평가(CPQRA)의 수행 흐름을 설명하시오.
【빈출/회차】 실제기출 10회 (CPQRA 흐름)
【근거】 [PSM 고시 제28조](https://www.law.go.kr/행정규칙/공정안전보고서의제출·심사·확인및이행상태평가등에관한규정) · [통설]

**모범답안** — PSM 고시 제28조 + [통설] CPQRA(Chemical Process Quantitative Risk Analysis):
> [고시 제28조①] 사업주는 단위공장별로 **인화성가스ㆍ액체에 따른 화재ㆍ폭발 및 독성물질 누출사고**에 대하여 각각 **1건의 최악의 사고 시나리오**와 각각 **1건 이상의 대안의 사고 시나리오**를 선정하여 **정량적 위험성평가(피해예측)**를 실시한다.
> [고시 제28조②] 시나리오는 「누출원 모델링」·「사고 피해예측 기법」·「최악의 누출 시나리오 선정」·「화학공장의 피해 최소화대책」 기술지침에 따라 작성한다.
> [통설] CPQRA 흐름: **① 사고 시나리오 식별**(HAZOP/What-if 등) → **② 빈도분석**(FTA·ETA·고장률 → 발생빈도) → **③ 결과분석**(누출원 모델링 → 확산 → 화재·폭발·독성 영향) → **④ 위험도 산정**(위험 = 빈도 × 피해) → **⑤ 위험도 평가·판정**(개인적/사회적 위험도를 허용기준과 비교) → 불충분 시 대책 보강·재평가.
> 🔗 함정(❌) **위험도 = 발생빈도 × 피해크기**. 최악(거리 최대)·대안(현실적·발생가능성 높음, §제2조29·30호) 시나리오 선정. 빈도=FTA/ETA, 결과=확산·영향 모델. 연계(→Q15-19 / 분류3 BLEVE·UVCE)
> 💡 **암기**: CPQRA 5단계 "시빈결산판" 💥 **시**험 끝 → **빈**털터리 → **결산** → **판**정 — 시(사고 시나리오 식별)·빈(빈도분석 FTA·ETA)·결(결과분석: 누출·확산·영향)·산(위험도 산정=빈도×피해)·판(허용기준 비교·판정)

### Q15-19. 정성적 위험성평가와 정량적 위험성평가를 비교하여 설명하시오.
【빈출/회차】 예상
【근거】 [PSM 고시 제2조](https://www.law.go.kr/행정규칙/공정안전보고서의제출·심사·확인및이행상태평가등에관한규정) · [통설]

**모범답안** — PSM 고시 제2조제6호 + [통설]:
> | 구분 | **정성적(Qualitative)** | **정량적(Quantitative)** |
> |---|---|---|
> | 결과 | 위험 유무·원인·등급(High/Med/Low) | **수치**(빈도·피해거리·위험도) |

### Q15-20. Fail Safe, Fool Proof, Tamper Proof의 개념을 설명하시오.
【빈출/회차】 실제기출 12회 (Fail Safe·Fool Proof·Tamper Proof)
【근거】 [통설] (안전공학 설계원칙)

**모범답안** — [통설] 본질적 안전설계 원칙:
> **Fail Safe(페일 세이프)** = 부품·시스템이 **고장 나더라도 안전측으로 작동**하도록 설계하는 원칙. 예) 전원 차단 시 밸브가 닫히는 **Fail-close**, 신호 상실 시 정지. 유형: Fail-passive(정지)·Fail-active(경보 후 일정시간 운전)·Fail-operational(다음 점검까지 기능 유지).
> **Fool Proof(풀 프루프)** = **작업자가 실수(오조작)해도 사고로 이어지지 않도록** 하는 설계. 예) 안전커버 없으면 작동 불가, 양수(兩手) 조작 버튼, **인터록**, 커넥터 오삽입 방지 형상.
> **Tamper Proof(탬퍼 프루프)** = 안전장치를 **임의로 해제ㆍ변경ㆍ무력화하지 못하도록** 하는 설계. 예) 봉인, 공구 없이 해제 불가, 안전밸브 임의조정 방지.
> **Fool Proof 종류 5** = ① **가드(Guard)**: 조절 가드ㆍ경고 가드ㆍ인터록 가드 ② **인터록 장치**: 기계식ㆍ전기식ㆍ유압식 ③ **트립(Trip)**: 신체 일부가 위험영역에 접근하면 기계 정지 ④ **오버런(Over-run)**: 스위치를 끈 후 관성운동ㆍ잔류전하를 감지하여 위험이 있는 동안 가드가 열리지 않음 ⑤ **밀어내기(Push-away)**: 가드 가동부가 열렸을 때 자동으로 비위험영역으로 신체를 밀어냄.
> **Fail Safe 예시** = 승강기 정전ㆍ정격속도 초과 시 **조속기** 작동 긴급정지, 장치의 **Redundancy(다중화)** 개념, **Bypass Line** 설치, 석유난로 기울어짐 시 자동소화, 철도신호 고장 시 적색신호 전환.
> **Tamper Proof 예시** = **휴즈(Fuse)**, (해제 시 작동 불능) 안전장치 봉인.
> 💡 **암기**: Fool Proof 5 "가인트오밀" 💥 **가인**(歌人)이 **트**로트 **오**디션 **밀**어붙여 — 가(가드)·인(인터록)·트(트립)·오(오버런)·밀(밀어내기)
> 🔗 함정(❌) **Fail Safe = 고장 시 안전측**, **Fool Proof = 사람 실수 방지**, **Tamper Proof = 임의 조작 방지**(셋의 대상이 다름: 고장/실수/고의해제). 연계(→Q15-22 / 분류1 안전장치)

### Q15-21. 하인리히와 버드의 재해발생 이론(도미노 이론·재해비율)을 설명하시오.
【빈출/회차】 예상
【근거】 [통설] (산업안전 재해이론)

**모범답안** — [통설] 재해발생 이론:
> **하인리히(Heinrich) 도미노 이론** — 재해는 **① 사회적 환경·유전적 요소 → ② 개인적 결함 → ③ 불안전한 행동ㆍ불안전한 상태 → ④ 사고 → ⑤ 재해(상해)**의 5단계 연쇄. **3단계(불안전 행동·상태) 제거**가 핵심(도미노 한 개를 빼면 연쇄 차단). **재해비율 1 : 29 : 300**(중상 1 : 경상 29 : 무상해사고 300).
> **버드(Frank Bird) 수정 도미노** — **① 통제 부족(관리) → ② 기본원인(기원) → ③ 직접원인(징후) → ④ 사고(접촉) → ⑤ 상해ㆍ손실**. 직접원인보다 **근본(관리)원인 제거**를 강조. **재해비율 1 : 10 : 30 : 600**(중상 1 : 경상 10 : 물적손실 30 : 무상해ㆍ무손실 600).
> 🔗 함정(❌) **하인리히 1:29:300 / 버드 1:10:30:600**(혼동 빈출). 하인리히=불안전행동·상태 제거 / 버드=관리(통제부족)·근본원인 강조. 연계(→Q15-22)
> 💡 **암기**: ① 하인리히 "사개불사재" 💥 **사**고뭉치 **개**가 **불** 질러 — **사**람도 **재**산도 끝 (1:29:300) — 사(사회적 환경·유전)·개(개인적 결함)·불(불안전 행동·상태)·사(사고)·재(재해) ② 버드 "통기직사상" 💥 **통**닭 **기**름 **직**화 — **사상**자 속출 (1:10:30:600) — 통(통제 부족)·기(기본원인)·직(직접원인)·사(사고)·상(상해·손실)

### Q15-22. 스위스치즈 모델·4M과 작업자실수분석(HEA)을 설명하시오.
【빈출/회차】 예상
【근거】 [통설] · [PSM 고시 제2조](https://www.law.go.kr/행정규칙/공정안전보고서의제출·심사·확인및이행상태평가등에관한규정)

**모범답안** — [통설] + PSM 고시 제2조제9호:
> **스위스치즈 모델(Reason)** — 사고는 여러 **방호벽(층)의 구멍이 일직선으로 정렬**될 때 관통하여 발생. 구멍 = **잠재적 실패(latent failure, 조직·관리 결함)**와 **능동적 실패(active failure, 현장 실수)**. → **다중 방호(심층방어)**로 구멍 정렬을 차단(LOPA와 통함).

> 🔗 함정(❌) 스위스치즈 = **잠재+능동 실패의 정렬**(조직요인 강조), 4M = Man·Machine·Media·Management. HEA는 **사람 실수**를 정량 분석. 연계(→Q15-17·Q15-21·Q15-20)

### Q15-23. HAZOP의 위험도 구분과 위험관리기준을 설명하시오.
【빈출/회차】 예상 (원문스캔)
【근거】 [KOSHA C-C-37-2026](#k-cc37)

**모범답안** — KOSHA C-C-37 §6.5 + §8:
> [§6.5(1)] 위험도는 사고의 **발생빈도와 강도를 조합하여 1에서 5까지** 구분한다(발생빈도 **3단계(상·중·하)** × 강도 **4단계(치명적·중대함·보통·경미)**).
> [§6.5 표5(강도)] **4(치명적)** = **사망, 부상 2명 이상, 재산손실 10억원 이상, 설비 운전정지 10일 이상** / 3(중대함) = 부상 1명, 재산손실 1억~10억원, 운전정지 1~10일 / 2(보통) = 부상자 없음, 재산손실 1억원 미만 / 1(경미).
> [§6.5(3)] **위험도를 결정할 때 발생빈도는 현재 안전조치를 고려하여 결정하나, 강도는 현재 안전조치를 고려하지 않는다.**
> [§8 표7(위험관리기준)] 위험도 **5(허용불가 위험)** = **즉시 작업중단**(작업을 지속하려면 즉시 개선) / **4(중대한 위험)** = **긴급 임시안전대책을 세운 후 작업하되 계획된 정비·보수기간에 안전대책 수립** / 3(상당한 위험) = 계획된 정비·보수 기간에 안전대책 / 2(경미한 위험) = 안전정보 제공 / 1(무시할 수 있는 위험) = 현재 안전대책 유지.
> 🔗 함정(❌) **빈도는 현재 안전조치 "고려", 강도는 "미고려"**(빈출 함정). 강도 4(치명적) 기준 수치 = **부상 2명 이상·재산 10억 이상·정지 10일 이상**. 위험도 **5=즉시 작업중단, 4=긴급 임시대책 후 작업**. 연계(→Q15-24·Q6-20)

### Q15-24. HAZOP 회의 진행 원칙과 결과 기록지 작성 방법을 설명하시오.
【빈출/회차】 예상 (원문스캔)
【근거】 [KOSHA C-C-37-2026](#k-cc37)

**모범답안** — KOSHA C-C-37 §5.4 + §6.4 + §6.6:
> [§5.4 회의 진행] (1) 팀 리더는 팀원이 모든 공정의 이탈 상황을 평가하도록 하고, **서기가 토의결과를 빠짐없이 서류로 작성**하도록 한다. (2) 서로 다른 의견이 있을 경우 **리더가 직접 결정하지 말고** 팀원에게 다시 묻거나 분야별 전문가 의견을 들어 조정한다. (3) **개선권고사항은 팀 전원의 동의가 필요하며 다수결에 의해 처리하지 않도록 한다.**
> [§6.4 기록지 구성] **이탈번호 → 이탈(가이드워드×변수) → 원인 → 결과 → 현재 안전조치 → 위험도 → 개선번호(우선순위) → 개선권고사항** 순으로 기록. 하나의 이탈에 원인이 여럿이면 **모두 기록**하고, 원인별로 결과를 각각 구분하여 기록.
> [§6.6(3) 개선권고 작성 3원칙] (가) **무슨 조치가 필요한가?**(무엇) (나) **어디에 이 조치가 필요한가?**(어디) (다) **왜 이 조치가 시행되어야 하나?**(왜).
> 🔗 함정(❌) 개선권고사항은 **반드시 전원 동의**(다수결 금지)가 핵심 함정. 리더는 의견 대립 시 **직접 결정 금지**. 기록지는 **이탈-원인-결과-현재 안전조치-위험도-개선권고** 구성, 작성 3원칙 = **무엇·어디·왜**. 연계(→Q15-23·Q15-1)
