📐 도식 고도화 — 13가지 방식 비교

같은 「안전장치(안전밸브·파열판·화염방지기)」 내용을 서로 다른 방식으로 시각화했습니다. 학습에 가장 도움되는 방식을 골라주세요.

규칙(법령) KOSHA 수치·기준 💡암기 ⚠️함정 면제·예외

색상 의미체계는 학습 보유율을 높입니다(연구: 시각+구조 결합 시 3일 후 보유율 10%→65%, 색상코딩 최대 60%↑).

A. 색상 플로우차트 의사결정Mermaid

설치→파열판 판단→검사주기 흐름을 색상 의미체계로. 규칙/수치/암기/함정을 색으로 구분.
학습 강점 분기·인과를 한눈에. 유지 텍스트 1줄=1노드, 자동레이아웃. 한계 노드 많아지면 복잡.

B. 서브그래프 그룹핑 분류Mermaid

설치·검사·배출을 박스(subgraph)로 묶어 구조를 시각적으로 구획.
학습 강점 대분류↔세부 관계. 한계 깊은 계층엔 부적합.

C. 손그림 스타일 암기친화Mermaid v11 handDrawn

안전밸브 작동 사이클을 손으로 그린 듯한 톤으로. 딱딱함을 줄여 친근하게.
학습 강점 부담 적은 비주얼, 노트 느낌. 한계 격식 자료엔 가벼워 보일 수 있음.

D. 타임라인 주기·이력Mermaid

검사주기(2·3·4년)처럼 시간/단계 정보에 최적.
학습 강점 주기·연혁·절차 순서. 한계 분기·관계 표현 약함.

E. 상태 다이어그램 동작원리Mermaid

안전밸브의 상태 전이(정상→과압→분출→재폐). 장치 작동원리 학습에 적합.
학습 강점 작동 메커니즘·조건. 한계 정적 분류엔 과함.

F. 클래스 다이어그램 속성비교Mermaid

안전밸브 vs 파열판의 속성을 구조적으로 대비.
학습 강점 두 개념의 속성 정리·상속관계. 한계 익숙치 않은 표기.

G. 사분면 차트 특성 포지셔닝Mermaid

압력방출장치를 2축(비용·반복성)에 배치해 상대 특성을 직관적으로.
학습 강점 비교·포지셔닝. 한계 정량근거 없으면 주관적.

H. 마인드맵 (현재 방식·기준선) 개관Mermaid

현재 사이트가 쓰는 방식. 비교 기준선.
학습 강점 빠른 개관. 한계 색상·강조 거의 불가(밋밋).

I. 인터랙티브 마인드맵 접기·줌Markmap

노드를 클릭해 접고/펼치고 줌·이동. 큰 토픽 전체를 담아도 탐색 가능. 마크다운 목록만으로 작성.
학습 강점 방대한 트리를 인터랙티브 탐색, 색상 자동. 한계 인쇄·정지캡처엔 불리.

J. 개념 관계 네트워크 조문 연계Cytoscape.js

§261~269 조문을 노드로, 상호 연계를 간선으로. 조문 간 관계망을 한눈에.
학습 강점 비계층 관계·연결 파악. 한계 노드 많으면 산만, 레이아웃 조정 필요.

K. 물리엔진 네트워크 관계 탐색vis-network

드래그 가능한 물리기반 그래프. 같은 조문망을 다른 엔진으로.
학습 강점 인터랙티브 탐색, 노드 끌어보기. 한계 위치 비고정, 캡처 비일관.

L. 인포그래픽 카드 요점 카드HTML/CSS

색상 카드 + 배지로 핵심을 압축. 가장 디자인 자유도 높음(수작업).
학습 강점 시각적 매력·요점 암기. 한계 자동화 약함, 카드마다 수작업.
